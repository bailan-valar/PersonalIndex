一个为个人和小团队设计的报销管理平台。它把发票识别、报销单整理、单据打印到状态跟踪串成一条顺畅的工作流，让原本繁琐的报销流程变得清晰高效。

## 项目背景

报销是件琐碎又容易出错的事：发票号码、金额、税额要逐项手填，费用和发票对不齐，打印单据时排版还要反复调整。我想要一个能把这套流程自动化的工具——发票上传即识别、报销单自动汇总、单据一键打印。Reim System 就是在这个想法下诞生的，覆盖了报销单从创建、整理、打印、审批到打款的全生命周期。

## 技术栈

- **前端框架**：Nuxt 3 + Vue 3 + TypeScript
- **UI 样式**：Tailwind CSS
- **数据库**：SQLite
- **ORM**：Prisma
- **PDF 处理**：PDF.js（客户端渲染）
- **数据可视化**：Chart.js
- **发票识别**：自动解析发票号码、类型、税率、金额、税额

## 主要功能

### 🧾 发票自动识别

- 上传发票文件即自动识别号码、类型、税率、金额、税额
- 无需手动填写，从源头消除录入错误
- 支持 PDF、PNG、JPG、JPEG，单文件最大 10MB
- 拖拽上传，操作流畅

### 📋 报销单一键打印

- 报销单信息自动填入，关联发票一键打印
- 优化多张发票打印排版，告别反复调整
- 内置 PDF 打印预览，所见即所得

### 📁 发票箱集中管理

- 集中管理所有发票，使用状态（未使用/已使用）一目了然
- 标签管理区分发票类型和使用场景（如替票）
- 批量标签、筛选、预览、下载一应俱全

### 💰 费用与发票自动计算

- 费用项目自动汇总报销单总金额
- 发票是否上传、金额是否匹配自动校验
- 五阶段状态流转：待整理 → 待打印单据 → 待审批 → 待打款 → 已完成

### 📊 可视化数据统计

- 首页仪表盘，报销单数量与金额统计
- 按状态、按类别的分布图表
- 直观掌握报销进度与财务状况

## 总结

这个项目聚焦于把报销这件"又碎又烦"的事彻底简化。发票自动识别省去了手填的繁琐，一键打印解决了排版的老问题，发票箱让每一张发票的去向都清清楚楚。它不追求大而全，而是把报销这一条核心流程做到足够顺手——这正是个人工具最该有的样子。
